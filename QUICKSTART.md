# Quick Start: Zero-Rules HR AI System

## Prerequisites

1. **SQL Server**: EmployeeDB at `localhost\SQLEXPRESS` with tables populated
2. **API Keys** (environment variables):
   - `GEMINI_API_KEY_1` and `GEMINI_API_KEY_2` (backup keys)
   - `GROQ_API_KEY`
   - `OPENROUTER_API_KEY`
3. **.NET 10.0** (for ASP.NET)
4. **Python 3.11+** (for FastAPI)
5. **ODBC Driver 17 for SQL Server** (installed on Windows by default or via `sqlserver-odbc` package)

---

## Startup Sequence

### Step 1: Install Python Dependencies
```powershell
cd d:\EmployeeSystem
pip install -r requirements.txt
```

Expected output:
```
Successfully installed fastapi uvicorn google-genai requests pyodbc
```

### Step 2: Start Python Backend (Terminal 1)
```powershell
cd d:\EmployeeSystem
uvicorn main:app --reload --port 8000
```

Expected output:
```
INFO:     Uvicorn running on http://127.0.0.1:8000
INFO:     Application startup complete
```

### Step 3: Start ASP.NET Frontend (Terminal 2)
```powershell
cd d:\EmployeeSystem
dotnet run
```

Expected output:
```
info: Microsoft.Hosting.Lifetime[14]
      Now listening on: https://localhost:5001
info: Microsoft.Hosting.Lifetime[15]
      Application started. Press Ctrl+C to shut down.
```

### Step 4: Test in Browser
Navigate to: `https://localhost:5001`

Go to: **Dashboard** → **AI Chat** (or equivalent AI panel)

## Verification Checklist

After starting both services:

- [ ] FastAPI responds: `curl http://127.0.0.1:8000/docs` → Swagger UI loads
- [ ] Database connection works: Check Python logs for `[DB]` messages
- [ ] /sql-agent endpoint works:
  ```powershell
  $body = @{ question = "Хэлтсээр ажилтан хэд байна?" } | ConvertTo-Json
  curl -X POST http://127.0.0.1:8000/sql-agent `
    -ContentType application/json `
    -Body $body
  ```
  Expected response: `{"sql":"SELECT DepartmentName, COUNT(*) ... "}`

- [ ] /sql-analysis endpoint works:
  ```powershell
  $body = @{ 
    question = "Ажилтан хэд байна?"
    results = '[{"EmployeeCount":125}]'
  } | ConvertTo-Json
  curl -X POST http://127.0.0.1:8000/sql-analysis `
    -ContentType application/json `
    -Body $body
  ```
  Expected response: `{"analysis":"Байгууллагад нийт 125 ажилтан байна."}`

- [ ] ASP.NET can reach Python:
  Check browser console or ASP.NET logs for HTTP 200 responses from `/sql-agent` and `/sql-analysis`

- [ ] Fallback chain works:
  If Gemini fails, check Python logs for `[AI] ✗ Provider: Gemini` then `[AI] ✓ Provider: Groq`

## Troubleshooting

### Python Error: "ODBC Driver 17 not found"
**Fix**: Install SQL Server ODBC drivers from:
```powershell
# Windows: Usually comes with SQL Server Client Tools
# Alternative: https://learn.microsoft.com/en-us/sql/connect/odbc/download-odbc-driver-for-sql-server
```

### Python Error: "pyodbc: Can't open lib"
**Fix**: Restart Python terminal or check ODBC driver installation
```powershell
pip list | findstr odbc
pip install pyodbc --upgrade
```

### ASP.NET Error: "Cannot connect to localhost:8000"
**Fix**: Verify Python FastAPI is running
```powershell
# Terminal 1:
netstat -ano | findstr 8000
# Should show: TCP    127.0.0.1:8000    LISTENING    [PID]
```

### No Results / "Өгөгдөл олдсонгүй"
**Debugging**:
1. Check generated SQL in logs
2. Run SQL directly in SSMS to verify it works
3. Check role security policy (if implemented)
4. Verify database data exists

### Gemini API returns 429 (quota exceeded)
**Expected behavior**:
- Python automatically tries Groq
- Check logs: `[AI] ✗ Provider: Gemini - 429: Too Many Requests`
- Should succeed with Groq or OpenRouter
